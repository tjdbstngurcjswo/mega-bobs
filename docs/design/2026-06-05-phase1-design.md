## 1. 디자인 토큰 (`shared.css` = 구현 기준)

```css
--bg:#F5F5F3          /* 페이지 배경 (뉴트럴 그레이) */
--surface:#FFFFFF     /* 카드 */
--surface-warm:#F7F7F5

--ink:#111111         /* 본문 */
--ink-2:#454545       /* 보조 텍스트 */
--muted:#8A8A8A       /* 캡션/메타 */
--line:#E3E3E0        /* 헤어라인 보더 */
--cream:#F5F5F3 / --cream-2:#9A9A9A   /* 다크 요소 위 텍스트 */
--primary:#E2C04C     /* 키컬러 — 저채도 옐로우 (유일한 액센트) */
--primary-deep:#C2A02E
--primary-soft:#FAF4D9
--accent-text:#997400 /* 라이트 배경 위 옐로우 텍스트용 */

--r:0 (전면)            /* 풀 스퀘어 — radius 금지 (예외: 슬롯 노브·풍선·토글 반원) */
--shadow: 0 1px 2px rgba(0,0,0,.05)   /* 플랫 — 헤어라인 + 미세 섀도 */
```

- 폰트: **Pretendard** (CDN). 스케일: 11.5/12/13/13.5/14/15/17/19/22/26/30/32px, 헤딩 weight 800 letter-spacing -0.02~-0.035em
- 테마: 라이트 단일 확정 — 다크/라이트 토글 제거 (2026-06-05 결정, next-themes 의존성 제거 가능)
- 헤더: 높이 64 다크 밴드(#111111), 로고(옐로우 11px 사각 + 메가밥스 워드마크) + 내비 5개(활성 = #242424 면 + 옐로우 밑줄) + 테마 토글(CSS 반원)

## 2. 화면별 스펙 (피그마/HTML 목업 대응)

### 홈 `/` — home-b.html (확정안)
- 정보 위계 (메뉴판 우선): ① 한 줄 히어로 "오늘 점심, 정하셨나요?" — 메뉴명 미언급 ② '식단' 보드(옐로우 헤더 → 날짜 네비 → 코스 행 3개 → 픽 한 줄(하단), 메뉴 리스트가 주인공) ③ 랜덤 추천 ④ 진입 리스트 — 이번 주 + 다음 주 탐색 가능 (§5)
- **콘텐츠 모델 주의**: '메인 메뉴'·'코스 대표명' 개념 없음 — 코스 A/B/테이크아웃별 동등한 메뉴 리스트뿐. 특정 메뉴를 대표로 세우는 카피·bold 금지
- 식단 보드 코스 행: 코스 라벨 + 합계 kcal + 메뉴 리스트(15px 600 잉크, 전 메뉴 동등 + 메뉴별 kcal)
- 주간 날짜 네비 (7일, 범위 ±1주): ‹(지난주까지 활성, 전전주 비활성+툴팁) + 요일 칩 7개 월–일(지난 요일 muted, 오늘 칩 = 옐로우 solid + '오늘' 표기, 주말 포함) + ›. 주 라벨·오늘 복귀 버튼 없음, 보드 타이틀 '식단' 고정. 식단 없는 날 선택 시 CLOSED, 다음 주 공개 전 COMING UP
- 식전 픽(보드 하단 행, 오늘 00:00~13:00 한정, 마감 후 행 제거): 메뉴를 다 본 뒤 픽 — 소형 버튼 4 + 우측 결과 텍스트('A 64% · 나가서 12% + 맛집 ›'). 상세: docs/superpowers/specs/2026-06-05-pre-pick-vote-design.md
- 히어로 카피: 특정 음식명 미지목 — "오늘 점심 확인하셨나요? / 내기는 30초면 끝납니다" (데이터에 '메인 메뉴' 개념 없음 — items는 평평한 {name, kcal}[] 배열, 2026-06-05 결정)
- 식단 보드: 옐로우 헤더(#E2C04C + 잉크 — 사이트 크롬과 위계 분리) + 행마다 COURSE 라벨(en) / 코스명 = 실데이터 라벨 `코스1/코스2/테이크아웃`(MenuCategoryLabel.ko) 20px 800 / 코스 합계 kcal(items 합산) / 음식은 · 구분 + 음식별 kcal 표기(숫자+kcal 단위 병기, muted, keep-all). '제육볶음 정식' 같은 합성 코스명·메인 음식 볼드 강조 사용 안 함
- **투표 (미니멀, 좌측 정렬, 13:00~자정 한정)**: 코스 행 하단 좌측에 썸 아이콘 버튼 2 + '89% · 27명'. 13:00 전엔 비노출(순수 메뉴판), 자정 후엔 결과 텍스트만. states.html 시간대 3프레임 참조
- 데스크톱 2컬럼: 좌 메뉴(556px), 우 사이드(300px: 랜덤 추천 그라디언트 카드 + 진입 2×2)
- 스크롤 없이 코스1 카드+투표가 1뷰에 들어와야 함 (기획 §4.1)

### 게임 허브 `/games` — games-hub.html

## 3. 접근성 & 비기능
- 컬러 대비: ink #111111 on #F5F5F3 ≈ 17:1. muted #8A8A8A는 캡션 한정. 라이트 배경 위 옐로우 텍스트는 #997400 사용
- 투표/게임 버튼 키보드 접근 가능, 슬롯 스페이스바 지원. 모션은 `prefers-reduced-motion` 시 결과 즉시 표시
- 아이콘 전략: UI 이모지 금지. 세리프 넘버링·이니셜·타이포 글리프·CSS 도형(토글 반원)으로 해결 — 별도 아이콘 셋 불필요

## 5. 메뉴 공개 정책 & 날짜 엣지케이스
- 열람 범위 (수정): 오늘 기준 ±1주 — 지난주 + 이번 주 + 다음 주. 전전주·다다음주 불가. 과거 날짜 맛평가는 결과 읽기 전용
- 업데이트: 매주 목요일 다음 주 식단 공개
- **투표 시간 정책 (경계 = 식당 마감 13:00)**: 픽은 00:00~13:00만 노출·선택(마감 시 행 제거) / 맛평가는 13:00~자정만 노출·선택(그 전엔 비노출, 자정 후엔 결과 읽기 전용). 픽과 맛평가는 공존하지 않음. RPC 서버 시각 재검증
- 날짜 네비: 요일 칩(월–금) + 주 이동 ‹ › — min 이번 주 월요일 / max 다음 주 금요일(공개 전엔 이번 주 금요일)

| 엣지케이스 | 처리 |
|---|---|
| 전전주 이동 시도 | 지난주 화면에서 ‹ 비활성 + 툴팁 "2주 전 식단은 제공되지 않아요" |
| 다음 주 — 공개 전 | COMING UP 빈 상태 "목요일에 공개돼요" + [이번 주로], › 는 활성 유지 |
| 다음 주 — 공개 후 | 정상 노출 + "다음 주" 라벨, 투표 미노출 |
| 다다음 주 | › 비활성 |
| 지난주·이번 주 과거 날짜 | 열람 가능, 픽 미노출, 맛평가 결과 읽기 전용 |
| 미래 날짜 투표 영역 | 버튼·결과 바 미노출 |
| 목요일 업데이트 지연 | COMING UP 유지(사용자에겐 장애 아님), 운영자 알림으로 인지 |
| 주말 접속 | CLOSED + 공개 후면 [다음 주 식단 미리 보기] CTA |
| 자동 블로그 | 지난주 비노출의 예외 — 과거 기록으로 열람 유지 |
