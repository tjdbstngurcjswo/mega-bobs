# UX Writing

한국어 UI 문구를 작성하거나 검수할 때 이 규칙을 따른다.

## 톤앤매너 원칙

- **어투:** 존댓말, 친근한 구어체 (`~이에요`, `~해요`, `~주세요`)
- **격식체 금지:** `-습니다`, `-합니다`, `-입니다` → `-이에요`, `-해요`, `-있어요`로 대체
- **주어:** 생략 가능하면 생략. 필요 시 `오늘은`, `영양사 선생님이` 등 구체적으로.
- **느낌표:** 긍정·활기찬 상황에만 (`만나요!`, `되세요!`). 에러·안내 문구에 사용 금지.

## 마침표 규칙

| 케이스 | 규칙 | 예시 |
|---|---|---|
| 단일 문장 | 마침표 **없음** | `오늘은 구내식당이 쉬는 날이에요` |
| 두 문장 연결 | 첫 문장 끝 마침표 **유지**, 마지막 문장 마침표 **없음** | `오늘 점심은 끝났어요. 내일 만나요!` |
| 쉼표로 문장 연결 | 쉼표 대신 마침표로 분리 | `주말이에요. 월요일에 만나요!` |

## 금지 패턴

```
❌ 지난 메뉴는 볼 수 없습니다.     → ✅ 지난 주 메뉴는 볼 수 없어요
❌ 다음 주 메뉴는 목요일에 공개됩니다. → ✅ 아직 준비 중이에요
❌ 잠시 후 운영 시작이에요.        → ✅ 잠시 후 운영 시작이에요  (마침표만 제거)
❌ 주말이에요, 월요일에 만나요!     → ✅ 주말이에요. 월요일에 만나요!
```

## 맞춤법 주의

- `맛평가` → `맛 평가` (띄어쓰기)
- `이용해주세요` → `이용해 주세요` (보조동사 띄어쓰기)
- `확인해주세요` → `확인해 주세요`
- `시도해주세요` → `시도해 주세요`

## 빈 상태(Empty State) 문구 패턴

```
label:  짧고 명확한 상태 레이블 (예: '휴무', '준비 중')
title:  따뜻하고 구체적인 한 문장, 마침표 없음
body:   행동 유도 또는 부연 설명, 마침표 없음 (선택)
```

### 현재 등록된 문구

| variant | label | title | body |
|---|---|---|---|
| `closed` | 휴무 | 구내식당이 쉬는 날이에요 | 인근 식당을 이용해 주세요 |
| `comingUp` | 준비 중 | 영양사 선생님이 메뉴를 고민하고 있어요 | 목요일에 다시 확인해 주세요 |

## 토스트 메시지

- 에러 토스트: 친근체, 마침표 없음
- 원인 + 해결책 패턴: `[원인]이에요. [해결책]` (두 문장일 때만)

```
❌ 메뉴 정보를 가져오지 못했습니다. 잠시 후 다시 시도해주세요.
✅ 메뉴 정보를 가져오지 못했어요. 잠시 후 다시 시도해 주세요
```

## 검수 절차

새 문구를 작성하거나 기존 문구를 수정할 때:

1. 격식체(`-습니다/합니다/입니다`) 없는지 확인
2. 단일 문장 끝 마침표 없는지 확인
3. 쉼표로 두 문장을 연결하지 않았는지 확인
4. 보조동사 띄어쓰기 (`~해 주세요`) 확인
5. 느낌표가 부정적 맥락에 쓰이지 않았는지 확인
